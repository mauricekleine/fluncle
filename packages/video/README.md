# @fluncle/video

Per-track social videos for Fluncle's Findings: 1080×1920 vertical clips that put one banger under the burning eclipse. This package is the **video machinery** every clip is built on. The rendered bundle in R2 is the durable archive.

The creative doctrine — the vehicles, the texture families, type staging, the research that grounds a scene in the track — does **not** live here. It lives in the **fluncle-video skill** at [`packages/skills/fluncle-video`](../skills/fluncle-video). That skill is the guideline an agent reads before it composes; this README is the reference for the kit it composes against. Read [DESIGN.md](../../DESIGN.md) (the visual canon, the Nostalgic Cosmos) and [VOICE.md](../../VOICE.md) (the copy canon) too. This README does not duplicate any of that doctrine; when it touches creative ground it points at the skill.

## The output contract

Each clip is authored as a self-contained Remotion composition dropped into the gitignored `src/remotion/workbench/<CompId>.tsx` (`export default` the component; the composition id is the filename). `root.tsx` auto-registers everything in `workbench/` via `import.meta.webpackContext`, so it is never hand-edited and a render leaves no tracked changes — the source is not a permanent repo artifact and there is no cleanup step. When the video ships, `bun run ship <trackId|log-id>` copies the exact render source into the upload bundle as:

```
out/<log-id>/composition.tsx
out/<log-id>/props.json
out/<log-id>/render.json
out/<log-id>/intent.json
out/<log-id>/metrics.json
```

Those files are uploaded to R2 beside the two masters (`footage.mp4` — clean square; `footage.social.mp4` — portrait baked-text), `poster.jpg`, `cover.jpg`, and `note.txt`. Everything past the two masters is **OPTIONAL**: `intent.json`, `metrics.json`, and the extra variant cuts (`footage.notext.mp4`, `footage.landscape.mp4`, `footage.landscape.social.mp4`) are packaged and uploaded ONLY when the file is present on disk — the CLI presigns only what the bundle actually contains, so a bundle without them uploads exactly as before. (The audio-less `footage-silent.mp4` is retired: surfaces derive a silent cut on the fly via an `audio=false` Media Transformation — see [docs/video-variants.md](../../docs/video-variants.md).) Keeping generated compositions out of the codebase stops them biasing future agents, while preserving enough source and analyzed props to rerender the same video when needed.

`render.json` records the composition id + source pointer, the props + intent pointers, the diversity-ledger `vehicle`/`grain`/`register` (the three diversity axes the upload step writes to `video_vehicle`/`video_grain`/`video_register`), the authoring provenance `model`/`reasoning`, and a **`variants`** provenance map keyed by output filename → the render flags that produced it. `register` is one of `abstract`/`representational`/`framed`; a missing `--register` WARNS at ship time but never blocks (it lands `null`). The two masters are the same composition + props rendered with different flags, so `variants` is what makes the bundle self-describing: re-rendering `footage.mp4` is `render(composition, props, variants["footage.mp4"])` (`{ aspect: "square", hideOverlay: true }`) and `footage.social.mp4` is `variants["footage.social.mp4"]` (`{ aspect: "portrait", hideOverlay: false }`), so a clean re-render from source reproduces the right cut instead of the portrait default. Writers derive the map from the shared `buildVariants()` helper (`src/remotion/variants.ts`); see [docs/video-variants.md](../../docs/video-variants.md).

Each composition remains **self-contained**: it inlines its own scene code, shaders, and scene composition, imports only the core surface from `../cosmos` plus Remotion/React/tokens as needed, and exports a named `React.FC<NostalgicCosmosProps>`. There are no shared vehicle components to assemble; a track owns its look top to bottom.

Two hard rules guard the output:

- **No generated track composition or rendered artifact is committed.** Temporary composition files, `out/`, and downloaded audio (`public/*.m4a`, `*.mp3`, `*.wav`) are local working/output state. The durable copy lives in R2 as part of the video bundle.
- **No static assets in renders.** Everything on screen is generated by code — shaders, SVG, CSS, transforms. Fonts are the only exception (loaded at module scope in `fonts.ts`). Plates and other bitmaps are banned; a frame must be reproducible from source alone.

## The core surface

`src/remotion/cosmos.ts` is the single public surface. Everything a track may import re-exports from here. It divides cleanly into **machinery** (tools) and **brand law** (rules baked into code).

### Machinery

- **`<ShaderLayer>` + `GLSL`** (`journey/shader-layer.tsx`, `journey/glsl.ts`) — the GPU workhorse. `ShaderLayer` renders a fullscreen-triangle fragment shader via raw WebGL, compiling once and pushing uniforms + drawing synchronously per frame. It injects a standard header (`u_time`, `u_res`, `u_progress`, `u_energy`, `u_bass`, `u_mid`, `u_treble`, `u_beatPulse`, `u_onsetPulse`, `u_audioHit`, `u_audioSwell`, `u_audioDrop`, `u_audioDisturbance`, `u_energyFast`, `u_bassFast`, `u_midFast`, `u_trebleFast`, `u_flux` (0..1 continuous transient/attack envelope, between-onset shimmer), `u_sub` (0..1 smoothed sub weight <60Hz — low-end pressure/mass), `u_kickHit` (0..1 near-raw transient-emphasized kick punch 60-150Hz), `u_snareHit` (0..1 near-raw transient-emphasized snare crack 2-5kHz), `u_air` (0..1 air band >5kHz — hat tails/cymbal wash), `u_downbeatPulse` (0..1, snaps on each bar downbeat and decays across the bar), `u_seed`, `u_palette[4]`, plus `ditherValue`/`dither8`) so a shader body can rely on it. The bands let a scene map different instruments to different elements — the music-driven win: the three broad bands (`u_bass` <150Hz, `u_mid` 150Hz-2kHz, `u_treble` >2kHz) plus the FINE bands (`u_sub` <60Hz, `u_kickHit` 60-150Hz, `u_snareHit` 2-5kHz, `u_air` >5kHz — the last three transient-emphasized MATERIAL signals, shared-normalized so cross-band loudness survives). `GLSL` is a library of composable function strings (`hash`, `valueNoise`, `simplexNoise`, `fbm`, `curlNoise`, `domainWarp`, `voronoi`, `dotField`, `paletteRamp`/`retint`, `polarFold`, `sdf`, `sdf3d`, `raymarch`, `filmGrain`, `grainFamilies`, `vignette`, `chromaticAberration`, `caustic`, `neuroWeb`, `swirlWarp`, `oklab` (`paletteRampOk`/`mixOklch`), `colorSpots`, `grainDisplace`, `bayer` (`bayer2`/`bayer4`/`bayer8`), `liquidMetal`, `tonemap` (`acesFilmic`/`reinhardJodie`/`liftGammaGain`), `noise3` (`valueNoise3`/`fbm3`/`voronoi3`/`curl3`)) you spread into a fragment shader ahead of `void main()`. Six of the newer fields/materials — `caustic`, `neuroWeb`, `swirlWarp`, `oklab`, `colorSpots`, `liquidMetal` — are vendored from **[Paper Shaders](https://github.com/paper-design/shaders) (Apache-2.0)** and retinted to canon; `tonemap`, `noise3`, `bayer`, and `grainDisplace` are Fluncle's own. `noise3` is the MONOTONIC third-dimension phase advance (feed a z that streams forward on `u_time` and a field roils in place instead of scrolling); `caustic`/`neuroWeb`/`swirlWarp`/`colorSpots` are liquid/organic field generators; `oklab`/`tonemap` are the perceptual-ramp + filmic soft-clip colour tools. `dotField` is the dense AA stipple/particle screen (flocks/swarms/halftone — dense, not sparse); `curlNoise`/`voronoi`/`domainWarp` are the fluid/cellular/marbled field helpers; `sdf3d`+`raymarch` are the 3D/volumetric pipeline. This is where grain and the palette ramp live — bake them into the shader (`filmGrain`, `paletteRamp`). `filmGrain` has two overloads: the legacy `filmGrain(col, uv, time, amount)` and a `GrainOpts` form (`filmGrain(col, uv, time, o)`) whose knobs (`scale`/`boilHz`/`clumpScale`/`clumpAmt`/`aniso`/`color`/`basis`/`seed`) the six `grainFamilies` presets package (`grainFineEmulsion`, `grainCoarseSilver`, `grainHalftone`, `grainChemicalDye`, `grainVhsScanline`, `grainDither`) — pick a family per video and set the AMOUNT to the composition (bias LOW toward near-silent — never fully off; grain is a signature every frame). There is no CSS `Grain` overlay; the recovered-footage grain is yours, in-shader, present and boiling every frame but with its amount and form the composition's call.
- **Opt-in bloom** — `<ShaderLayer bloom={{ threshold, intensity, radius }} />` adds a real multi-pass emission glow (bright-pass + separable blur + add, single-frame and deterministic). Off by default. Bloom the vehicle's OWN hot material, never a glued-on glow-orb (doctrine 1). The composite adds RGB ENERGY only and **preserves the scene's alpha** (`gl_FragColor = vec4(scene.rgb + bloom * intensity, scene.a)`), so a localized alpha-composited layer (an orb/glow that fades to true zero) keeps its soft edge under bloom instead of printing an opaque rectangle (the quad law). No cross-frame feedback exists (Remotion renders frames independently), so motion-trail effects must use in-shader history, not GPU feedback.
- **Texture inputs (artwork-as-texture)** — `<ShaderLayer textures={{ <name>: <src> }} />` uploads each image as a `sampler2D <name>` the shader can sample, plus a `float <name>AspectRatio` (`width / height`) for aspect-correct sampling. `<src>` is an https URL (pass `track.artworkUrl` straight in) or a `staticFile()` path / bare `public/` filename. Every image loads once (`crossOrigin="anonymous"`) behind Remotion's `delayRender()`/`continueRender()`, so no frame is captured before the textures are ready; a load failure `cancelRender`s with a clear message. NPOT-safe (`CLAMP_TO_EDGE` + `LINEAR`, no mipmaps) so any artwork size works in WebGL1; Y is flipped on upload (`UNPACK_FLIP_Y_WEBGL`) so the canonical `uv = gl_FragCoord.xy / u_res` (y-up) samples the image UPRIGHT. Units are assigned deterministically in sorted name order; textures re-upload on context restore; remote URLs work headless. The loader is keyed on the map's CONTENT (a stable string of sorted `name=src` pairs), so a per-frame-recreated `textures` object never reloads or recompiles — keep the values frame-derived.
- **`glsl3` (WebGL2 / GLSL ES 3.00 opt-in)** — `<ShaderLayer glsl3 />` switches the context to `webgl2`, emits the header as `#version 300 es`, and expects the body to write the injected `out vec4 fragColor;` (declare no colour output yourself) and sample textures with `texture(...)`. The injected uniform NAMES are unchanged. There is NO silent fallback — if webgl2 is unavailable the error overlay shows. Absent (default) stays WebGL1 with byte-identical behavior. Purpose: verbatim GLSL3 lifts. `OES_standard_derivatives` (`fwidth`/`dFdx`) is built in under GLSL3 and auto-enabled under WebGL1 when the host supports it.
- **Audio hooks** (`hooks/`) — `useAudioReactivity`, `useBeat`, `useDownbeat`, `useOnset`, `useEnergy`, `useBass`, `useMid`, `useTreble`, `useFlux`, `useSub`, `useKick`, `useSnare`, `useAir`, and the lower-level `sampleCurve`/`smoothCurveAtFrame`. Pure, deterministic, audio-reactive: each computes from `useCurrentFrame()`/`fps` and the `audio.*` arrays only. `useAudioReactivity` is the default bus (exposes `energy`/`bass`/`mid`/`treble` + `*Fast` + `hit`/`swell`/`drop` + `flux` + `sub`/`kickHit`/`snareHit`/`air` + `beat`/`downbeat`/`onset` + `peakTimeMs`): audio disturbs the material and may move the picture too — motion only through a smoothed signal (Motion law, doctrine 7), never the raw beat. The drop envelope's peak defaults to `audio.dropMs` (the analyzer's detected drop) when present, else the loudest energy sample; explicit `reactivity.drop.peakTimeMs` still wins. The canonical named audio uniforms (`u_audioHit`/`u_audioSwell`/`u_audioDrop`, `u_bassFast`/`u_energyFast`/`u_midFast`/`u_trebleFast`, `u_flux`, `u_sub`/`u_kickHit`/`u_snareHit`/`u_air`, and `u_beatPulse`/`u_onsetPulse`/`u_downbeatPulse`) are pushed into the shader BY NAME; `u_audioDisturbance` is the only entry in the returned `uniforms` bag (the silent-zero alias class was closed — every uniform a shader declares by name exists in the header and is pushed by name).
- **The journey clock** — `useJourney(options?)` (`journey/use-journey.ts`). The shared narrative timeline: returns `{ progress, phase, phaseProgress, arc }`, where `phase` is `"depart" | "travel" | "arrive"` and `arc` is the eased 0..1 a scene travels along. Pure and frame-derived.
- **`@remotion/three` — escape hatch, not wired in.** `@remotion/three`, `@react-three/fiber`, and `three` are installed and available if a concept genuinely needs a full 3D / React-Three-Fiber scene, but **nothing in the core uses or enforces them** — the default surface is the raw-WebGL `ShaderLayer` above (leaner, deterministic, headless via ANGLE). Reach for R3F only when even `sdf3d`+`raymarch` inside `ShaderLayer` can't express the idea; you own the determinism + grain if you do.

### The fixed layer + scene-led palette (rules that live in code)

There is **no imposed look** here — no `Grain` overlay, no `Starfield`. Texture/grain (the Light-Years degradation) and any background (a star drift, motes) are the agent's own, baked into its shader via `GLSL.filmGrain`. What the code DOES enforce is the fixed information layer + the warm-dark/legibility/scene-led-palette guarantees:

- **`<TypePlate>`** (`primitives/type-plate.tsx`) — the FIXED informational type system every video shares: the identity block (Artist — Title, with the release credit `Label (Year)` beneath) lower-left, the telemetry block (Found date + Log ID) upper-right, prescriptive enter/exit timing that clears before the drop. The provenance line degrades gracefully (label-only / year-only / nothing). Drop it in once with scene-derived `ink`/`dimInk`; nudge `identityInSec`/`telemetryInSec` onto a beat when the music asks. Compositions never hand-place the facts.
- **`<FloatingType>`** (`primitives/floating-type.tsx`) — the underlying typography primitive (`brandMark`, `trackLine`, `meta`, `logId`, `body`) with the **ink-halo legibility guarantee** baked in, plus the em dash in `Artist — Title`, the `Found Jun 4` tabular UTC date format, and the One Voice font split (Oxanium for marks/numerals, system sans for copy). `TypePlate`/`CloseCard` consume it; use it directly only for `body` text beyond the plate. It formats only what it is given; it never adds punctuation.
- **`<CloseCard>`** (`journey/close-card.tsx`) — the mandatory close card: small dim tagline, then the selector signature as the loudest type moment, revealed as a staggered two-beat sign-off at its default lower-left home (the TypePlate identity anchor). Pass `palette` so both inks match the composition — no fixed gold; the signature ink follows the scene like all other type. Drive it from the `"arrive"` phase's `phaseProgress` via the `progress` prop. The legacy `arc` prop is **accepted but IGNORED** (the arc trap, now fixed in code): the reveal used to fall back to `arc ?? progress`, and callers reach for the journey's GLOBAL `arc` (`useJourney().arc`, already non-zero through most of the clip), so the sign-off printed mid-clip — the reveal is now driven ONLY by `progress` (`arc` stays for runtime-compat with archived compositions). Every video ends here.
- **`paletteMix`** (`palette-mix.ts`) — derives the palette from the artwork swatches, **scene-led**: the only imposed rule is Warm Dark (the field stays a warm near-black, nudged toward the artwork's darkest). `accent` is the artwork's OWN most-chromatic swatch (no gold lean), `glow` is that accent's hot light, `ink` stays cream. Gold appears only if the artwork is genuinely warm — it is never imposed. Falls back to the brand palette only when no swatches are supplied.
- **Color/font helpers** — `hexToRgb`, `rgbToHex`, `withAlpha`, `mix`, `luminance`, `warmth`, `saturation` (`color.ts`); `OXANIUM`/`OXANIUM_STACK`/`loadOxanium` (`fonts.ts`); the `NostalgicCosmosProps`/`CosmosTrack`/`CosmosAudio`/`CosmosPalette`/`EnergySample` contract types (`types.ts`).

#### The hard laws, restated

These are guarantees the code makes; a track must not defeat them.

1. **Warm Dark** — `paletteMix` keeps the field a warm near-black (nudged toward the artwork's darkest); `accent`/`glow`/`ink` are scene-led, no imposed gold.
2. **The contrast guarantee** — `FloatingType` keeps every type role legible over the scene.
3. **Smooth motion (doctrine, not a component)** — any background/field you author has a smooth baseline drift (frame-derived); audio MAY bend its motion (travel, flow, convergence), but only through a smoothed envelope, never the raw per-beat transient that would snap it (Motion law, doctrine 7). (The core no longer ships a `Starfield`.)
4. **The quad law** — a `ShaderLayer` must never show its quad as a hard rectangle. A **full-bleed background** fills the frame (opaque, a real value into the corners) and so leaves no edge. A **localized layer** (orb/glow) fades **color AND alpha to true zero** before its bounds — a shader that returns nonzero outside its shape prints a rectangle (the printed-rectangle incident). The inverse over-correction is the **porthole**: a tight radial vignette on a full-bleed field crops the 9:16 canvas into a circle — keep background vignettes gentle (radius ≳ 1.0).

## The pipeline

Run from `packages/video` (or `bun run --cwd packages/video <script>`). Use **bun**, never npm/pnpm/yarn.

```
bun run social:preview <trackId> [--skip-render] [--composition <Id>] [--composition-source <file>] [--duration-ms <10000-30000>]
```

The full local loop: fetch track → resolve a preview clip → download + **two-pass loudnorm** to −14 LUFS (`I=-14:TP=-1.5:LRA=11`, `linear=true` — measure pass then one fixed-gain re-encode, so a preview never pumps mid-clip and the normalization is deterministic) → transcode to an AAC `public/<trackId>.m4a` → analyze audio (honest bpm + confidence, beat grid + bar downbeats, onsets, drop + drop candidates, energy/bass/mid/treble + the fine sub/kick/snare/air bands, flux) → extract artwork palette → assemble inputProps → write `out/<trackId>.props.json` → (unless `--skip-render`) bundle + render `out/<trackId>.mp4` → write `out/<trackId>.render.json`. A **bounded m4a cache** keeps only the 8 most-recently-downloaded preview `.m4a`s under `public/` (an opportunistic sweep each run); `ship` additionally deletes a shipped track's own `public/<trackId>.m4a` once R2 holds the durable copy.

- **`--skip-render`** — stop after writing the props JSON. The fast loop for checking analysis, palette, and the assembled contract before paying for a render.
- **`--composition <Id>`** — render a specific registered composition. Required for renders; there is no default track composition.
- **`--composition-source <file>`** — explicitly record the source file that produced the render. If omitted, the pipeline records `src/remotion/workbench/<Id>.tsx`, matching the auto-registration contract.
- **`--duration-ms <10000-30000>`** — pick the clip length off the waveform (end on a clean beat). Drives `durationInFrames`.

Other scripts: `bun run studio` (Remotion Studio, the live scrub loop), `bun run typecheck` (`tsgo --noEmit`, the quality check for any change here).

The per-hop curve-builders the analyzer relies on (`computeBands`, `onsetEnvelope`, the normalizers) live in `src/pipeline/audio-curves.ts`, shared so both the render path and the set path use the same kernel.

### Set analysis (Fluncle Studio, Unit B)

- **`bun src/pipeline/analyze-set.ts <set-audio> [out.json]`** (`src/pipeline/analyze-set.ts`) — turns a long DJ-set audio file into a `StudioEnvelope`: full-length energy/bass/flux curves (decimated to a ~100ms display hop; the 20ms analysis stays internal) plus a ranked list of candidate **drop** windows the operator vets in the editor. The decode is a STREAMING ffmpeg pass at 11025Hz (a 48-min set never materializes as a whole-WAV buffer + float array). The picker is NEW logic vs the render path's single-20s-window kernel: drop-novelty scoring (a quiet breakdown → a loud bass slam, weighted by the re-entry flux), **local** (windowed) max-normalization so quieter good drops survive, spacing-deduped peak-picking (~35s apart), a **local** per-peak tempo/phase snap (a single global BPM drifts on a multi-tempo set), and a musical pre-roll so the drop lands just inside the window. Candidates are loudness-rise guesses (`kind: "drop"`), not certainties — a soft re-entry false-positives. `bpm` is `null` on a multi-tempo set.

### Evaluation scripts (the gates)

- **`bun run detect-beat-pull <trackId|video>`** — the motion-reversal gate: measures the picture's short-lag translation reversal (lurch-then-snap) and exits non-zero on a fail. The single hard gate on the Motion law (doctrine 7).
- **`bun run judge:metrics <trackId|video> [--json] [--intent <file>] [--allow-flash]`** (`src/pipeline/analyze-motion.ts`) — the deterministic-metrics script: writes `out/<trackId>.metrics.json` and carries **THREE hard gates** from one structural pass — **flash-safety** (WCAG 2.3.1 / ISO 9241-391: a coherent >3/sec luminance or saturated-red strobe on a sliding 10° sub-window), **beat-pull** (short-lag motion reversal — the Motion law), and **arc/deadness** (`scoreArc`, the structural TWIN of beat-pull on the LONG timescale: it samples 5 anchor frames across the clip and fails when the picture never structurally reorganizes — `wholeClipChange < ARC_FLOOR` of 0.29 → a frozen field / looping wallpaper / near-static bars). Any hard gate exits non-zero and blocks ship (`hardPass` is the roll-up); the report also carries advisory coupling + intent-vs-actual reads. `--allow-flash` overrides the flash gate only after an eyeball (never on a ship candidate), and `--intent <file>` points it at the render-intent file to check actual-vs-intended.
- **`out/<trackId>.intent.json`** — the render-intent spine (schema `fluncle.render-intent/1`, frozen in `src/pipeline/intent.ts` as `type RenderIntent`): authored by the agent at concept time and shipped into the bundle as `intent.json`. It declares the `vehicle`, `textureFamily`, `register`, `arcSource`, `motionModel`, `dropMs`, `climax {form, colour, atMs}`, and the per-band → element `bindings[]` (`{band, element, axis, intendedStrength}`) the metrics and the judge check the render against; a missing intent is warn-and-stub in v1 (ship writes a generated stub).
- **`bun run validate:intent <intent.json> [--json]`** (`src/pipeline/validate-intent.ts`) — the STRICT render-intent linter (the authoring-time counterpart to the runtime validator): precise per-field errors on `schema`/`trackId`/`vehicle`/`concept`, the enum fields (`textureFamily`/`register`/`arcSource`/`motionModel`), `dropMs`, `climax`, and every `bindings[]` entry (`band ∈ ALL_BANDS`, `axis ∈ ALL_AXES`, `intendedStrength ∈ {subtle, strong}`). Exit `0` valid, `1` invalid, `2` usage/read error.
- **`bun run judge:diversity <posterPathOrLogId> [--neighbours N] [--strict] [--json]`** (`src/pipeline/judge-diversity.ts`) — the advisory diversity check: measures a poster's structural distance from its recent published feed neighbours (fetched from the public `/api/tracks` feed, newest-first). Distance is structure-dominant — a 9-bin Sobel edge-orientation histogram (weight 0.60, the recolor-invariant discriminator) blended with an HSV colour histogram (0.20) and luma-contrast (0.20); `DIVERSITY_MIN = 0.35`, and only the IMMEDIATE neighbour is the hard constraint. **Advisory by default** (exit 0 + a verdict); `--strict` makes an immediate-neighbour clone exit non-zero. It does not block ship on its own.
- **`bun run calibrate [--footage] [--json]`** (`src/pipeline/calibrate.ts`) — the calibration harness that keeps the gate thresholds RE-EARNABLE as operator verdicts accumulate. It reads the labeled corpus `calibration/verdicts.json` (entries: `{ logId, dimension: "arc"|"diversity"|"judge", verdict }`), replays the matching deterministic gate on each entry's PUBLIC artifacts — `poster.jpg` pairs for `diversity` (offline), `footage.social.mp4` for `arc` (downloaded only with `--footage`) — and prints a gate-vs-operator table flagging disagreements with a threshold-nudge suggestion. `judge` rows are a documented coverage BOUNDARY (printed, never scored). Exits non-zero only when a SCORED row disagrees with its label.
- **`bun run lint:composition <composition.tsx> [--json]`** (`src/pipeline/lint-composition.ts`) — the LAUNDERING-PROOF static motion lint. It scans the composition source (JS + GLSL) for the Motion-law violations: `no-constant-base` (an audio uniform drives a translation term with no clock term), `audio-exceeds-clock` (the audio coefficient ≥ the constant-clock coefficient on a translation term), and `audio-tainted-phase` — the laundering guard that follows a JS-side rename: it taints `uniforms={{ name: <expr> }}` object literals bound to audio values (`audioRx.*`, bus fields like `.swell`, `use*()` hooks, or a var holding one), one hop of indirection, so a renamed audio value re-enters the GLSL scan as an audio token and can't be smuggled onto a coordinate. Exits non-zero on any finding — it catches at author time what `judge:metrics` would fail after a full render.
- **`bun run ship <trackId|log-id> [--vehicle <tag>] [--grain <family>] [--register <abstract|representational|framed>] [--model <provider/model>] [--reasoning <level>]`** (`src/pipeline/ship.ts`) — packages the bundle. `--register` is the third diversity axis (a missing one WARNS but never blocks). Ship **refuses a draft-only output** (a half-res `out/<trackId>.draft.mp4` alone is not shippable) and requires a Log ID. It renders the clean square crop-source master in-process (`{ aspect: "square", hideOverlay: true }`) and **caches it at `out/<trackId>.square.mp4`**, reusing that on re-ship instead of re-rendering. Extra variant cuts (`footage.notext.mp4`/`footage.landscape.mp4`/`footage.landscape.social.mp4`) are packaged only when their source file is already on disk.

### Props contract

Every producer (pipeline, Studio defaults, the agent) satisfies `NostalgicCosmosProps` from `src/remotion/types.ts`:

```ts
type NostalgicCosmosProps = {
  track: CosmosTrack; // + optional track.features (spectral summary) & track.contextNote/track.texture (context-note character) — both creative fuel
  audio: CosmosAudio; // file, startMs, durationMs (10000–30000), bpm, bpmConfidence? (0..1), beatGrid[], downbeats? (bar downbeats, ms), onsets[], dropMs? (detected drop, ms), dropCandidates? ({timeMs, score}[]), energyCurve[], bassCurve[], midCurve[], trebleCurve[], fluxCurve? (continuous transient/attack), subCurve?/kickCurve?/snareCurve?/airCurve? (fine bands, shared-normalized), rawDynamicsHint? (per-band pre-normalization crest hint)
  palette: CosmosPalette; // background, ink, accent, glow, swatches[]
  seed: number;
};
```

`durationInFrames` is derived from `audio.durationMs` in `root.tsx` via `calculateMetadata`, so the clip length always matches the audio. The pipeline writes real props to `out/<trackId>.props.json`; `ship` copies those props to `out/<log-id>/props.json`.

**Honest BPM + drop/downbeat detection (the DSP layer).** `bpm` is now **honest** — never clamped into the D&B family (the old `[160,185]` clamp is gone) — and carries a `bpmConfidence` (0..1, from the autocorrelation peak prominence + harmonic-comb agreement), so a low confidence or an out-of-family tempo is a signal to verify by ear, not a silent correction. The analyzer also ships `downbeats` (every 4th beat at the kick-scored bar phase → `u_downbeatPulse`), `dropMs` (the primary detected drop — a quiet-breakdown→loud-slam novelty, re-entry-flux weighted — which becomes the drop envelope's default peak), and `dropCandidates` (all scored candidates, score-descending, as fuel for choosing an alternate climax). The FINE bands — `subCurve` (<60Hz), `kickCurve` (60-150Hz transient-emphasized), `snareCurve` (2-5kHz transient-emphasized), `airCurve` (>5kHz) — are **shared-normalized** across each other, so cross-band loudness survives (a quiet snare stays quiet relative to the kick) instead of each band being independently stretched to fill 0..1. Every new field is optional so old `props.json` on disk and Studio defaults still typecheck (a missing curve reads 0 / no pulse — prior behaviour).

**`track.features`** (optional) is the enrichment skill's track-level spectral summary — `{ centroidHz, subBassRatio, highRatio, midFlatness, onsetRate }` — fetched from the public API and passed through as CREATIVE FUEL: it steers the vehicle/texture/band-mapping at concept time, never per-frame reactivity (that is `audio.*`). Absent until a track is enriched, so treat it as optional. This is the enrichment → video handoff; the two analyses run alongside each other (track-level direction vs per-frame animation), neither replaces the other.

**`track.contextNote` + `track.texture`** (optional) are the SECOND direction input — the finding's distilled `context_note` (firecrawl FACTS → a small LLM; see `apps/web/src/lib/server/observation.ts`) and, split out of its trailing `Texture:` line, 3–6 sensory/scene/mood pointers. The note is INTERNAL (admin-gated), so the pipeline reads it via the CLI (`fluncle admin tracks context <id> --json`, no re-fetch — the same read the observe sweep uses) rather than the public `/api/tracks`. Pure CREATIVE FUEL, **never on-screen text**: where `features` says HOW a track sounds, `texture` says WHAT it evokes (orchestrated/atmospheric/foundational) — it leans the vehicle/texture-family/palette/scene at concept time and INFORMS, never templates (see the fluncle-video skill §"Metadata must visibly matter"). Absent until a finding is context'd, so treat it as optional; a bare `packages/video` checkout with no `fluncle` on PATH simply gets no note. (Forward tie-in, not a dependency: the aliveness RFC's `intent.json` spine is the natural home for these declared direction inputs once it's built; for now they ride `track` directly.)

### Encode settings rationale

`h264`, **`crf: 23`** under a **VBV bitrate cap** (`encodingMaxRate: "32M"`, `encodingBufferSize: "64M"`), **`colorSpace: "bt709"`**, with **lossless PNG intermediate frames**, `x264Preset: "slow"`. The grain is high-entropy, so it saturates whatever bitrate you give it — crf alone is the wrong lever (crf 31 was blocky; uncapped crf 20 rendered cleanly but ballooned a 20s clip to ~200MB). The cap is what bounds size: 32 Mbit/s × 20s ≈ 80MB, comfortably under the **100MB** input ceiling for Cloudflare Media Transformations (so the master can drive an on-the-fly web rendition off R2). PNG intermediates stay — JPEG baked banding in before the h264 pass. crf and the cap are tuned by eye on each render batch; grain is load-bearing (the Light-Years Rule), so the cap can't go so low it re-introduces blocking. Grain SCALE matters here too: coarser grain (`GrainOpts.scale > 1`, the coarse-silver family) has fewer high-frequency transitions and compresses BETTER than 1px speckle, so fine + heavy is the worst case for the cap — reach for a coarser family or a lower amount before fighting the encoder.

- **`colorSpace: "bt709"` is explicit** because Remotion 4.0.x still defaults to `'default'` (full-range `yuvj420p`), which drifts in brightness/colour against what TikTok/YouTube assume on re-encode and changes how the grain reads. bt709 outputs limited-range `yuv420p` (verified on the file). Set it; don't rely on the version default.
- **Grain preservation is bounded by crf + the cap — that's the whole toolbox.** Remotion exposes only `crf` and `x264Preset`; there is **no** way to pass an x264 `tune` (e.g. `tune=grain`/`film`) or custom x264 params. `overrideFfmpegCommand` is discouraged AND does not reach the per-chunk video encode (verified: spliced `-tune`/`-x264-params` leave the embedded settings at preset-slow defaults). So don't chase encoder grain-tuning here — it's not available.
- **The grain SMEAR risk is the TRANSCODE path, not this render.** Re-compressing an already-encoded grainy master at a lower bitrate (the phase-1 file-size pass) makes h264 smear the grain temporally — it reads slow/dead. This render path (PNG → x264) is clean. To shrink an oversized clip without killing grain, **re-render from source** (`composition.tsx` + `props.json` are in the R2 bundle), don't re-transcode the master.

**Draft mode (`--draft`) — verify-only, never shippable.** Checking motion / reactivity / "does the drop land" needs the whole clip rendered with audio, but the ship encode above is deliberately slow. `bun run social:preview <trackId> --composition <CompId> --draft` writes `out/<trackId>.draft.mp4` at **half resolution** (`scale: 0.5` → 540×960; the GLSL shader + bloom are per-pixel, so ~4× fewer pixels is the big win), `x264Preset: "veryfast"`, jpeg intermediates, `crf: 28`, and no VBV cap — measured **~6× faster end-to-end** (7s vs 44s on a stipple-flock comp; the gap widens on heavier shaders). **`fps` is NOT lowered** — it's exactly what a motion check needs, and the beat-pull gate runs on a draft for a FAST directional read. Caveat: half-res can UNDER-report reversal on fine-stipple / live-grain clips (the churn only fully manifests at full res — a clip read 0.15 on the draft vs 0.23 on the full), so the draft gate is directional, not authoritative — confirm the PASS on the full render. And a draft CANNOT show whether the load-bearing grain reads or blocks (half-res + jpeg hide it), so the final grain judgment also needs the one full render — and `ship` refuses to package a draft-only output. The ship encode above is untouched by `--draft`.

### GPU / ANGLE note

GPU shaders need a real GL context, and the renderer is **one source of truth** — `glRenderer()` in `src/pipeline/gl.ts`, read by `selectComposition`/`renderMedia` (`chromiumOptions.gl`) and by `remotion.config.ts` (so `remotion still`/`render` inherit it — no `--gl` flag needed). It defaults to **ANGLE** (Metal on Apple Silicon — verified "ANGLE Metal Renderer"); a GPU-less host (the remote render box) exports `FLUNCLE_GL=swangle` (SwiftShader-on-ANGLE: software, visually identical, slower).

### Determinism rules (non-negotiable)

A frame must be a pure function of its inputs; Remotion renders the same frame many times across processes.

- No `Math.random()` and no wall-clock time (`Date.now()`, `new Date()` for "now") inside a composition. Use Remotion's `random(seed)` and frame-derived values (`useCurrentFrame()`, `fps`) only.
- Derive every seeded layer from the `seed` prop (and stable sub-seeds) so a track always renders identically.
- Format dates from `track.discoveredAt` in UTC (`FloatingType` `meta` does this) so renders don't depend on host timezone.
- Audio reactivity comes only from the `audio.*` arrays through the hooks, never from decoding audio at render time.
- Shader uniforms are deterministic too: `u_time = frame / fps`, `u_progress` from the clip, audio uniforms from the curve hooks — never wall clock or `Math.random`.
- Fonts load once at module scope (`fonts.ts`), not inside the frame function.
