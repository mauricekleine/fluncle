# Fluncle Sprites — the Sprite System

The single doctrine for every pixel sprite in the Galaxy. Today that is the **Galaxy game** set (`@fluncle/sprites`' `galaxy` collection — the ship, Earth, the asteroid, the roadster, the UFO), the **void** set (the `void` collection — the black hole at the empty coordinate on the 404 page: the accretion disc, the event-horizon ring, and the traveller's Discman falling in), and the **probes** set (the `probes` collection — the /about survey fleet: the probe and the telescope, the instruments that measure and never speak); any future set joins the same family. They are never separate art — **one family**, and this skill is what keeps them one family. (The Earth-overworld prop set and the per-surface icon map that once fed the internal `/sprites` gallery were pruned with that page; git history keeps their specs.)

**The core truth (write it on the wall):** cohesion is a _set_ property, not a per-sprite one. It comes from a small list of **fixed rails applied identically to every sprite** — one perspective, one light, one shared palette — _not_ from polishing each sprite alone. And **no prompt alone produces a true sprite**: every AI render is a high-res _mock_ that MUST pass through a deterministic key → crop → resize → quantize post-step. AI gets you most of the way; the deterministic remainder is non-negotiable.

The **north stars** are the retired `web.privacy` padlock and `web.docs` book (in git history at `packages/sprites/assets/earth/`): recognizable in half a second, a light cream body that POPS off the dark plate, one or two brand accents (the gold shackle, the red ribbon), a crisp 1 px dark outline, no halo. Every sprite matches their density, contrast, palette discipline, and clarity — detailed HD pixel art (the reference tier), never the starved low-res that mushes the form.

## The fixed rails (constants — never vary; restate ALL of them in every prompt)

Constraints do NOT carry between generations — an un-repeated rail silently drifts — so the prompt scaffold (`TAIL` in the script) restates every one, every time.

| Rail            | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Output size** | **The sprite's logical width** (`w`, ~26–46 px — square-ish ~40, tall subjects ~26–30, wide ~44–46). This is the **read-quality lever**: ~16 px starves the form, ~40 px gives the outline + a few shading steps room to land. Stored at that size; the browser upscales it crisply with `image-rendering: pixelated`. No fixed large export — the file IS the sprite.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Perspective** | **A LOCKED three-quarter view with a PINNED FACING** — every object faces the SAME way: its front toward the **lower-left**, receding to the upper-right (a top-right→bottom-left axis, like a tidy sprite sheet), seen from slightly above, lit from the upper-left. Locked set-wide; never mix angles _or_ facings (a vague "front-left" lets the model drift ↘ vs ↙ — pin the diagonal). 3/4 shows an extra plane → more readable + dimensional than flat; depth comes from the light, not strict geometry. **Gameplay exceptions** (vertical, head-on, symmetric, NOT angled — because the player moves along that axis): a **takeoff craft** is a **front view pointing straight up** via `pf()` / `TAIL_FRONT`; the **playable ship** is a **rear view pointing up/away** (the player's POV) via `pr()` / `TAIL_REAR`. **Celestial exception** — a radially SYMMETRIC body (a black hole, a ring) has no front to pin, and 3/4 squashes a circle into an ellipse, so it stays a true circle seen **face-on** via `pc()` / `TAIL_FACE` (the centered-orb motif, DESIGN.md); only the camera changes, the one upper-left light holds. |
| **Light**       | **ONE key light, upper-left**, set-wide. No second light. No pillow-shading (concentric value rings with no light logic).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Footprint**   | Subject fills most of the frame, centered. The post-step **crops to the bounding box**, so framing is automatic — just prompt "centered, fills most of the frame."                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Outline**     | A **deterministic 1 px dark contour** (`add_outline`, in the shared darkest warm-black `#090a0b`) wraps the silhouette in the post-step — the crisp edge every reference sprite carries, and the single biggest "pop" lever after resolution. Guaranteed, not luck-of-the-render.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Shading**     | Hard pixel edges with **cel-shaded volume** (a few clear value steps). **No anti-aliasing blur, no gradient banding, no dither, no drop shadow, no scene/ground.** One object only. (The prompt asks for the detail and bans the blur.)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Key BG**      | A solid flat **pure magenta `#FF00FF`** background. Keyed out, then **cropped** to the subject. NO white-buffer outline — that left a halo; the crop trims the edge instead.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Colors**      | A **per-sprite SUBSET** of the Sprite Palette — only the colors that sprite should use (a mailbox = creams + blacks + reds, **no gold**). Composed from the named ramps → [references/palette.md](references/palette.md).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |

## The color system (the make-or-break)

Pixel sprites blend into our dark plates when their biggest surface uses a dark token. The fixes, stated precisely:

- **Pop is VALUE contrast, not hue.** The dominant mass must sit at a distinctly **higher lightness** than the dark plate it lands on. Two hues at the same value still blur — _lightness_ carries the form. Default the dominant body to the **cream ramp** (`CREAMS`).
- **Per-sprite palette SUBSET — the cohesion lever AND the anti-creep guarantee.** Quantize each sprite to ONLY the colors it should use, never the whole 17-color set. A mailbox has no gold in its subset, so gold _cannot_ creep into a cream highlight. Compose the subset from the named ramps; **`CREAMS + BLACKS` is always the spine** (so every sprite shares the same lightest + darkest endpoints — that shared start/end is what makes separate sprites a family by construction), then add only the accents that sprite earns.
- **Brand colors ride as ACCENTS, never the body.** Eclipse Gold + Re-entry Red appear at only ~10–20% of the area, reserved for one identity highlight (the One Sun Rule still governs gold — one note, never a field). State it in the prompt as "<accent> only on X."

Full ramps + hex + the per-sprite-subset rule in [references/palette.md](references/palette.md). It descends from `DESIGN.md`'s Nostalgic Cosmos and matches the canon ramp in `apps/web/src/game/palette.ts`, `docs/galaxy-sprites.md`, and `@fluncle/sprites`' `SPRITE_PALETTE`.

## The form rules (clarity first)

1. **Silhouette-first.** A sprite must be recognizable as a **solid shape** (squint/blur test) BEFORE any color. If color is what makes it legible, the form is broken — change the subject framing. (`web.home`/monolith failed this; the mailbox and rocket pass.)
2. **Recognizable in half a second.** Pick the 2–3 function-defining features; abstract everything else away. Make the defining feature **bold and frame-filling** — the failure mode is "a cream box with small stuff inside," which reads as a generic panel (the early switchboard/fusebox/status all fell into this); lead with the one shape that names the thing (a gauge is a circle, a clapperboard is its striped bar).
3. **Every pixel earns its place.** Stray single pixels read as pure noise at this scale; the crop + quantize curb most, but eyeball for orphan specks (even the rocket carries a few) and prefer a subject that won't shed them.

## The AI generation pipeline (the deterministic spine)

Treat the AI render as a MOCK on flat magenta; the clean sprite comes from the post-process. Scripted at [scripts/generate_sprite.py](scripts/generate_sprite.py). Pure PIL — no numpy.

1. **Per-sprite spec.** Add an entry to `SPRITES`: `{ w: <logical width>, palette: <subset from the named ramps>, prompt: p("<subject> + its colour story") }`. The colour story names the body tone and pins each accent ("a small red flag, red only on the flag"; "no gold").
2. **Shared scaffold, every prompt.** `p(subject)` appends `TAIL` — the rails restated in full: **detailed 16-bit pixel-art icon**, a **uniform THREE-QUARTER VIEW with the facing pinned (every object's front toward the lower-left, receding to the upper-right — the same for all)**, single centered object, no scene/ground, **cel-shaded volume in a few value steps with strong dark edges**, one upper-left light, **solid flat pure magenta `#FF00FF` background**, and the explicit bans (no blurry AA, gradient banding, dither, drop shadow, text, logos, UI). Gameplay craft use the vertical head-on cameras instead — `pf()` (`TAIL_FRONT`, a takeoff craft pointing up) or `pr()` (`TAIL_REAR`, the player ship from behind). (Do NOT say "8-bit / chunky / low-res" or "flat front" — both bias the model away from the dimensional HD look.)
3. **Generate** via Gemini (`gemini-3.1-flash-image`, `:generateContent`, header `x-goog-api-key`, body `{"contents":[{"parts":[{"text":prompt}]}]}` — NO `generationConfig`/`responseModalities`, which 400s). One sprite per call.
4. **Post-process (deterministic):** key magenta → transparent → **crop to the subject's bounding box** (trims any edge halo) → **resize to the logical width** (LANCZOS) → **quantize every opaque pixel to the per-sprite SUBSET** (nearest by squared distance; drops near-transparent pixels so the alpha edge stays clean) → **`add_outline`** wraps the silhouette in the 1 px dark contour.
5. **Save** the small PNG at `packages/sprites/assets/<collection>/<id>.png` (the `@fluncle/sprites` canonical home; the web build mirrors it to `public/`). The PNG name == the sprite id.
6. **Wire it** in `packages/sprites/src/index.ts`: add the id to `SPRITES.<collection>` so the manifest stays true.
7. **PILOT one sprite end-to-end and eyeball it next to its family BEFORE fanning out** — view the PNGs together upscaled with nearest-neighbor, and compare to the source of truth (the north-star look), not just "it ran." A subtle regression a pilot waves through multiplies across the whole set.
8. **For a stubborn subject, render variants and pick (or keep several).** When one subject resists (the rocket took three passes), add 2–3 scratch entries with different takes in one run, eyeball them together, then copy the winner onto the real id (or promote several as a family — `launch_rocket` + `rocket_riveted` + `rocket_capsule`) and drop the scratch entries.

## Run it

```bash
GEMINI_API_KEY="$(op read 'op://<vault>/GEMINI_API_KEY/credential')" \
  UV_CACHE_DIR=/tmp/uv-cache uv run --with pillow \
  python packages/skills/fluncle-sprites/scripts/generate_sprite.py <id> [<id> …]
```

(Run in a background terminal so `op` can prompt for approval; the key inlines into the subprocess env and never lands anywhere on disk.) No args = regenerate every entry in `SPRITES`. Then mirror to the web and view it in place: `bun apps/web/scripts/copy-sprites.ts`, open the Galaxy.

## The gate (a sprite is NOT done until all pass)

- **Silhouette read** — recognizable as a shape at small size.
- **Value-contrast** — the dominant cream mass clearly lighter than the dark plate; doesn't blur into it.
- **Clean key** — NO white halo / border (the magenta key + crop did its job).
- **Palette** — only the per-sprite subset; accents ≤~20% area; gold only where the colour story put it.
- **One light, upper-left**; no pillow-shading.
- **No orphan pixels, no merged forms.**
- **Crisp upscale** — display only with nearest-neighbor (`image-rendering: pixelated`), never bilinear/bicubic.
- **Taste-gate alongside the family** — view the new sprite next to the existing set (in the Galaxy itself, or the PNGs side by side upscaled nearest-neighbor); does it belong?
