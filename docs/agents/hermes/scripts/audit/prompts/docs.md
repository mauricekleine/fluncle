# Tonight's domain: Docs freshness

Docs drift silently — nobody's typecheck fails when a doctrine doc keeps describing a file, flag, or route the code renamed six PRs ago, so the doc quietly starts lying to every future reader and every agent that trusts it. Tonight you audit the written layer against the shipped one: doctrine vs code, links that still resolve, `AGENTS.md` staying principle-level, and prose that follows its own rules. Prefer **fixing** mechanical drift (a dead link, a renamed path, a wrong function name) and **filing** the larger restructures.

## The hunt

**1. Doctrine-doc ↔ code drift (highest value — this is where trust rots).** The canon-adjacent docs describe how the system actually works: `docs/surfaces-doctrine.md`, `docs/track-lifecycle.md`, `docs/admin-shell.md`, `docs/naming-conventions.md`, `docs/video-variants.md`, `docs/local-database.md`, `docs/set-video.md`, `docs/helm.md`, and the agent docs under `docs/agents/` (`enrichment-agent.md`, `hermes-agent.md`, `newsletter-agent.md`, `observation-agent.md`, `note-agent.md`). Take each concrete claim a doc makes — a file path, a function/export name, a CLI command, a flag, a route, a table/column, a script in a `package.json` — and **verify it against the code** (grep for the symbol, stat the path). A claim whose referent no longer exists or was renamed is drift: **fix** the wrong name/path (it's mechanical and verifiable); **file** a doc whose whole described flow has diverged (that's a rewrite, a judgment call).

**2. Dead internal links + broken relative paths.** Across every Markdown file — `docs/`, the root canon, package READMEs, `packages/skills/**/SKILL.md`, the `docs/agents/hermes/**/README.md` cron docs — resolve each relative link and `](./…)` path target against the tree. A link pointing at a moved or deleted file is a safe, high-confidence **fix** (repoint it to the real location, or drop it if the target is gone). Watch specifically for files that recently moved (e.g. planning docs now living under `docs/planning/`, RFCs under `docs/rfcs/`).

**3. `AGENTS.md` stays principle-level + its docs index resolves.** `AGENTS.md` MUST stay principle-level (per its own Docs rules): no repeatable-workflow sprawl, no secret-bearing operator commands, no `op://` paths / hostnames / IPs / tailnet names / topology (this repo is public). Flag creep where a concrete operator command belongs in a runbook or skill instead. Then walk the entire **canon docs list** at the bottom of `AGENTS.md` and confirm **every** cited path still exists — a broken index entry is a mechanical **fix**.

**4. Stale non-canonical brainstorms that contradict shipped code.** `docs/rfcs/*.md`, `docs/planning/` (roadmaps), and any `docs/*-brief.md` are explicitly non-canonical per `AGENTS.md` — where they deviate from the code or from canon (`DESIGN.md`/`PRODUCT.md`/`VOICE.md`), the code wins. Do **not** rewrite an RFC to match the code (that erases the brainstorm's intent) — **file** a note where a shipped RFC/roadmap now contradicts reality, so a human decides whether to retire it. A shipped-and-superseded RFC that no doc points to being done is a merge/delete candidate → file it.

**5. README accuracy vs the package it documents.** Each `README.md` (`packages/video`, `packages/live`, `packages/media`, `apps/helm`, `apps/extension`, `apps/ssh/watchdog`, `docs/socials`, and the root `README.md`) should match its package's real scripts, exports, commands, and layout. A README listing a `bun run` script the `package.json` no longer has, an export that moved, or a directory that was renamed is a **fix**.

**6. Markdown prose on single logical lines.** `AGENTS.md` mandates prose paragraphs on single logical lines — no hard line breaks mid-sentence, no reflow-to-column. In any Markdown you touch tonight, a mid-sentence hard wrap is a mechanical **fix** (rejoin to one logical line). Don't reflow files you're not otherwise editing — keep the diff scoped to real findings.

**7. Duplicate / parallel explanations.** Two docs explaining the same flow divergently is a maintenance trap (`AGENTS.md`: prefer deleting, merging, or linking stale docs over adding a parallel explanation). A trivial cross-link ("see X") is a safe **fix**; a genuine merge/dedupe of two overlapping docs is a restructure — **file** it.
