# Tonight's domain: Design canon integrity

Fluncle's whole product is one image made walkable ŌĆö a traveler's logbook read _through_ the interface into the cosmos, dark-only, cover-led, one sun. The canon (`DESIGN.md`, "The Nostalgic Cosmos", plus `PRODUCT.md`) is not taste, it is the spec: the design system is what makes the archive feel inevitable rather than styled, and every drift toward a SaaS dashboard, a second accent, or a hand-rolled primitive erodes it. Tonight you audit `apps/web`'s surface against that canon ŌĆö the mechanical, verifiable rules first, the judgment calls filed.

## The hunt

**1. Design-system adherence + no headless primitives in feature code (highest value ŌĆö mechanical and verifiable).** The Shadcn source of truth is `@fluncle/ui` (`packages/ui/src/components/*.tsx`), imported into feature code as `@fluncle/ui/components/<name>` ŌĆö see `apps/web/components.json` (`ui` alias, `iconLibrary: phosphor`). `@base-ui/react/*` belongs **only inside** those `packages/ui` wrappers; a direct `@base-ui` import anywhere under `apps/web/src` is a finding (`rg -n "@base-ui" apps/web/src`). So is a local re-implementation or alias/wrapper of a component that already exists in `packages/ui/src/components/` (button, dialog, dropdown-menu, popover, select, tabs, tooltip, ŌĆ”) ŌĆö use the canonical generated export, don't hand-roll it. If a needed Shadcn component is genuinely absent, the fix is to add it via the Shadcn CLI, not to reach for a primitive; that's a file, not an auto-edit.

**2. Iconography split ŌĆö never crossed.** Interface icons (actions, status, nav, affordances) come from Phosphor (`@phosphor-icons/react`); third-party brand/platform logos come from `simple-icons`, rendered through `BrandIcon` (`apps/web/src/components/brand-icon.tsx`) or the wrappers in `apps/web/src/components/platform-icons.tsx`. A Phosphor glyph standing in for a brand mark (a Phosphor "spotify"/"youtube" lookalike, a hand-redrawn logo) is a finding ŌĆö quote the official mark. Confirm brand SVGs stay `fill="currentColor"` so they inherit surface ink and obey the One Sun budget.

**3. Token consistency ŌĆö no raw-value drift.** The palette, radii, and the shared `--grain-tile` live as CSS variables in `apps/web/src/styles.css` (`--eclipse-gold`, `--starlight-cream`, `--deep-field`, `--stardust`, `--dust-line`, `--reentry-red`, `--nebula-violet`, `--radius-*`). Feature `.tsx`/CSS should reference those variables, never a raw hex duplicate of a canon color (`rg -n "#[0-9a-fA-F]{6}" apps/web/src` ŌĆö third-party theming shims like Scalar in `docs.api.tsx` are the known exception; a _new_ duplicate in an app surface is drift). Enforce the visual laws these tokens carry: **The One Sun Rule** (Eclipse Gold only for primary action / focus / identity, ~10% ceiling; Nebula Violet is the _sole_ sanctioned second light and only in the live-set callout), **The Warm Dark Rule** (no cool grays / blue-tinted darks), **The Through-the-Glass / One Pane Rule** (depth is translucency over the fixed backdrop ŌĆö a `box-shadow` used for elevation, or glass stacked on glass, is a finding), and **The Ignition Rule** (a control that _dims_ on hover is wrong).

**4. Public app stays dark-only, cover-led, centered, quiet.** Sweep the public routes (`apps/web/src/routes/index.tsx`, `log.$logId.tsx`, `artist.$slug.tsx`, `mixtapes*`, `radio.tsx`, `stories/`) for the `PRODUCT.md` anti-references, verbatim: SaaS-dashboard chrome, bright streaming-app clones, generic landing-page hero sections, oversized marketing copy, glassy card stacks, and decorative gradients that ignore the cover art (every gradient must derive from the eclipse palette). No light theme, no second accent hue. The tracklist is the page; the cover art is the hero.

**5. Accessibility ŌĆö the Legible Sky, keyboard, reduced-motion.** Text meets WCAG AA (4.5:1) against what is _actually_ behind it, grain and glass included (**The Legible Sky Rule**) ŌĆö a pane over the bright sun region must get more opaque, never the text dimmer. Interactive rows and links keep real keyboard access (the Track Row in `apps/web/src/components/track-row.tsx` is one focusable link, not a div-with-onClick). Every ambient motion is gated to `prefers-reduced-motion: no-preference` and grounded under `reduce` (the 72s cosmos drift and ~48s sun-bloom breath in `styles.css`, the float/press transitions); an ungated animation, or a JS matchMedia listener that forgets the `reduce` branch, is a finding.

**6. Voice on any copy you touch (defer deep copy to VOICE.md).** If a fix touches a user-facing string ŌĆö a button label, empty state, error, toast, meta text ŌĆö it follows `VOICE.md` and the right surface register. Off-voice or marketing-toned copy that isn't in scope for a design fix is a file, not a rewrite.
