# Tonight's domain: Test coverage

Tests are part of "done" тАФ `deploy:gate` (the Cloudflare deploy boundary, `package.json` тЖТ `format:check && lint && typecheck && test`) runs **every** package's suite, so an untested surface is a silent regression waiting to land on `main`. Fluncle's convention is a colocated `*.test.ts` next to the unit it exercises (`apps/web` runs `vitest run --coverage`; `packages/registry`, `packages/contracts`, `apps/cli`, and most others run `bun test`). Tonight, hunt behavior that shipped **without** a focused test, coverage nets that have gone slack, and flaky patterns that will eventually fail a deploy build for reasons unrelated to the change under test.

## The hunt

**1. Recently-added units with NO focused test (highest value тАФ this is the silent-regression class).** Walk the last ~20 commits (`git log --name-only`) for new/changed source under `apps/web/src/lib/server/**`, `apps/web/src/lib/**`, `packages/**/src/**`, and `apps/cli/src/**`, and for each one check whether a sibling `*.test.ts` exercises it. Known gaps to confirm and close: `apps/web/src/lib/server/artists.ts` and `apps/web/src/lib/artist-socials.ts` (the fresh artist-relationship work тАФ `artist-resolution.ts` has a test, these do not). **Prefer FIXING** by adding a focused test for a clearly-untested _pure_ unit (a parser, a normalizer, a DTO shaper, a slug/notation helper) тАФ a real, durable fix you can run and verify. **FILE** the ones that need heavy DB/network mocking or a whole harness тАФ that is a larger coverage effort, not a one-file add.

**2. The build-fail coverage nets still guard what they should.** The four exhaustiveness scaffolds in `apps/web/src/lib/server/` тАФ `orpc-coverage.test.ts` (public routes), `orpc-admin-coverage.test.ts` (admin routes), `orpc-auth-coverage.test.ts` (every contract op sits at exactly one explicit auth tier), `orpc-naming.test.ts` (`verb_noun` shape). Confirm no PENDING allow-list has gone stale (a name that no longer maps to a real route), that the pending lists are still _shrinking_ not growing, and that a newly-added route/op is actually caught rather than silently slipping a carve-out. These are the tests that make "a new endpoint with no contract" a build failure тАФ a hole here is high-impact.

**3. Missing edge-case / error-path coverage on public & admin contracts.** Where a contract test asserts only the happy path, look for the un-covered branch that matters: the 401/403 auth-tier rejection, the rate-limit boundary, the not-found, the malformed-input `ORPCError`, the empty/duplicate case. Add the missing assertion when the unit and its mocks already exist in the test file; file it when the setup is substantial.

**4. Flaky patterns.** Real wall-clock timing (`await new Promise((r) => setTimeout(r, тАж))` тАФ e.g. `route-masking.test.ts`, `push.test.ts`) that should use `vi.useFakeTimers()`/`vi.setSystemTime()` like `admin-auth.test.ts`, `capture-queue.test.ts`, `enrich-queue.test.ts` do; unseeded `Math.random`/`Date.now` driving an assertion rather than a documented mock (contrast the _intentional_ `tracks-radio.test.ts` mock, which is fine); order-dependence between tests sharing mutable module state; and any `apps/web` file whose first test could cold-import the `./orpc` graph without inheriting the 20s `testTimeout` from `vitest.config.ts` (the 5s default false-fails on a loaded build box тАФ that's why the timeout is raised). Fix the clear-cut ones; file a suite-wide pattern.

**5. Hollow or disabled tests.** Assertions that don't actually assert (a computed value never `expect`ed, a `try/catch` that swallows, an `expect` with no matcher), and any `.only`/`.skip`/`it.todo` left in (none at last sweep тАФ a fresh one is a finding). A test that always passes is worse than no test: it reads as covered.

## Where to look first

`package.json` (`deploy:gate`) ┬╖ `apps/web/vitest.config.ts` (the 20s `testTimeout` + the ratchet coverage thresholds) ┬╖ `apps/web/src/lib/server/orpc-coverage.test.ts` ┬╖ `apps/web/src/lib/server/orpc-admin-coverage.test.ts` ┬╖ `apps/web/src/lib/server/orpc-auth-coverage.test.ts` ┬╖ `apps/web/src/lib/server/orpc-naming.test.ts` ┬╖ `apps/web/src/lib/server/artists.ts` ┬╖ `apps/web/src/lib/artist-socials.ts` ┬╖ the colocated `*.test.ts` across `apps/web/src/lib/**` and `packages/**/src/**`.
