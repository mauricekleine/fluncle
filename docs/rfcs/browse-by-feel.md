# RFC: Browse-by-feel — the galaxies become real: sonic clusters over the MuQ embedding space, and a public lens to wander them

**Status:** Draft — divergent research, a taste pass, and a 3-role adversarial review (staff engineer, design/canon, product/scope) are baked in, and the operator's five rulings are ratified (2026-07-10). Ready for handoff.
**For:** a fresh build session or a small team of worktree sub-agents.
**Canon/authority:** `docs/track-lifecycle.md` (the enrichment architecture and the vibe-placement retirement), `apps/web/src/db/schema.ts`, the oRPC contract layer (`packages/contracts/src/orpc/**`), `@fluncle/registry`, `DESIGN.md` / `VOICE.md` / `LORE.md` / `PRODUCT.md`. This document is planning, not spec; the codebase and canon win on conflict, and this RFC is pruned once built.

## The standard (definition of done)

The whole thing, automated end to end, tested and documented — no manual per-finding step anywhere in the loop. The operator's touches are editorial and structural acts by design (naming a galaxy, triggering a split, blessing a remint), never plumbing. Every unit below ships complete: schema + ops + cron + admin gate + public lens + CLI + SSH + registry fan-out + the galaxy-reader swap + docs. Tests and doc updates are acceptance criteria, not follow-ups. The only "not yet" items are the future consumers listed under "does NOT build" — honest phasing (this RFC builds the object they will read), not deferral. Dangling threads this build ties off: every galaxy reader across `/log`, the OG card, radio, the CLI, and agent discovery moves from the dead vibe-quadrant derivation to the real map; the retired-but-still-writable `vibe_x`/`vibe_y` path goes; the `track-lifecycle.md` "solar systems" naming collision (it appears on BOTH lines 33 and 76) gets fixed; and the unbuilt Phase 2 the pruned audio-embedding RFC promised finally ships.

## 0. Summary / the reframe

- **The operator's ruling collapses the naming question: there is no new noun — the clusters ARE the galaxies.** The k=9 operator-named sonic clusters REPLACE the four vibe-bucket galaxies (Solar/Nebular/Lunar/Astral) and inherit the word. The galaxies stop being fiction and become the real, sound-derived map. Because the name carries over, most existing `track.galaxy` readers keep their surface untouched and simply get richer data behind them.
- **The unifying simplification: the galaxy is one first-class object.** A stable-ID, operator-named cluster entity living in the shared layer (a `galaxies` table + `tracks.galaxy_id` + public read ops + a registry surface). The browse lens, the `/log` prose, the OG card, radio, the CLI, the SSH terminal, and every future consumer (themed radio, the game) merely _read_ it. Never let the web lens own the cluster's identity — that is the mistake that would foreclose the game and radio.
- **Stability by construction: the nightly run is assignment-only.** Each night the cron assigns findings to their nearest stored centroid and recomputes each centroid as its members' mean — one bounded step, IDs stable because no relabeling step exists to go wrong. Full k-means fits run ONLY inside operator acts (cold start, split, remint). This is deliberate: a nightly full re-fit — even warm-started — can never emit an empty cluster (sklearn relocates an emptied centroid to a far point mid-fit), so a dying galaxy's bookmarked ID would silently teleport across the map; assignment-only makes "empties → retired" actually reachable and the fixed-point claim literally true.
- **Stability is coupled to naming, so map structure only changes by an operator act.** A split mints an unnamed child only the operator can name anyway, so the trigger and the naming belong to the same hand: the cron surfaces the evidence (per-cluster coherence, size), the operator pulls the trigger. No autonomous growth machinery, no nightly tunables.
- **Compute follows the established box pattern.** A nightly deterministic `--no-agent` `fluncle-cluster` sweep on rave-02 beside `fluncle-embed`, sub-second CPU, writing per-finding assignments via the generic agent update path and the map via one transactional admin-tier op — with the IDs and handles minted server-side (the box never mints identity).
- **The lens is "Close in sound" scaled from a row to a map.** Cover-led galaxy pages you wander — an anti-SaaS, browse-not-scroll stance synthesized from DESIGN.md's Don'ts, the VOICE.md web register, and the genre argument ("an absolute label is an argument; a relative placement is answerable") — never genre chips or filter pills.
- Decomposition: five slices — (1) schema + ops + the sweep's CLI subcommands, (2) the cluster engine, (3) the operator naming gate, (4) the public lens + the `/log` swap, (5) reach surfaces + fan-out + the galaxy-reader swap. Slice 1 unblocks everything; 2–4 build in parallel after it, but go-live is strictly 2 → the operator names the full initial map → 4.

## 1. Context & goals

Every published finding carries a 1024-d MuQ audio embedding in `tracks.embedding_json` (`apps/web/src/db/schema.ts:94`), written by the on-box `fluncle-embed` cron and already consumed by `get_similar_findings` (`apps/web/src/lib/server/tracks.ts:764`) for the `/log` "Close in sound" row (`apps/web/src/routes/log.$logId.tsx:421`). The pruned audio-embedding RFC (git `144f1cc4`, removed in `957fdd83`) shipped that Phase 1 and promised Phase 2: "periodic k-means → a lens grouping the archive by sonic family (name the clusters once they're stable)." The clusters never shipped. Meanwhile the four vibe-bucket galaxies (`apps/web/src/lib/galaxies.ts`) went functionally dark for new findings: manual vibe-tagging was retired (audio can't learn the mood axis — coin-flip, per the signal test in `track-lifecycle.md` §"Vibe placement (retired)"), so `vibe_x`/`vibe_y` are NULL for every new finding and the galaxy-guarded surfaces silently drop for them — while still rendering for the ~45 old placed findings, which makes those branches live code, not dead code (see the reader swap in Unit E). This RFC replaces the quadrant derivation with the real map: every embedded finding gets a galaxy, old and new alike, so those same surfaces come back to life with honest data.

Goals, honestly calibrated:

- **In reach:** a stable, named, sound-derived map of the archive; a public browse lens with per-galaxy indexable pages; a fully automatic nightly assignment refresh that never reshuffles the map; the galaxy in each `/log` page's prose and JSON-LD, now driven by sound instead of the dead quadrants. All deterministic engineering over data that already exists.
- **Outside our control:** whether search engines and AI crawlers actually surface the galaxy pages (we ship the structure — `MusicPlaylist` JSON-LD, sitemap, llms.txt — the citations are theirs to give); whether the k-means families keep _feeling_ right as the corpus grows (the operator's ear is the judge; the split and remint acts are the levers).

## 2. Unit A — the cluster engine (algorithm, stability, the box cron)

### The space and the metric

MuQ vectors are L2-normalized at embed time (`apps/web/src/lib/server/embedding.ts:64-70` documents this and defends against the exception), so Euclidean k-means on them IS cosine/spherical k-means — the exact metric `get_similar_findings` ranks with.

**Rejected: HDBSCAN.** It emits a noise label (`-1`) — a product dead end (a finding with no galaxy has no map cell to wander to), and density methods degrade on dense 1024-d embeddings (worst on 5/6 text-embedding datasets in benchmarks, driven by noise assignment — appendix). Every finding lands in exactly one galaxy; k-means gives the hard partition.

**Rejected: per-run auto-k (silhouette/gap re-selection).** Free-picking k every night is the _cause_ of churn (the gap statistic is simulation-based and non-deterministic; silhouette and gap conflict). k is structural, and structure only changes by an operator act.

**Rejected: the obvious nightly full re-fit + Hungarian label alignment.** The panel's staff-engineer review killed it with sklearn's actual behavior: a full `KMeans.fit` can never emit an empty cluster — its Lloyd loop relocates an emptied centroid to a high-inertia far point (`_relocate_empty_clusters`), i.e. an arbitrary different sonic region — and with equal k the post-hoc Hungarian matching is _perfect_, so the teleported centroid MUST inherit some prior stable ID. Both the warm start and the alignment fail together, producing exactly the bookmark-breaking reshuffle the feature forbids.

### The nightly run: assignment-only, a fixed point by construction

The `fluncle-cluster` sweep never runs a clustering fit. Each night it:

1. Loads the stored centroids (the map) and the embedded corpus.
2. Assigns each finding to its nearest centroid (cosine) — new findings join the map here; boundary findings may migrate marginally.
3. Recomputes each galaxy's centroid as the L2-normalized mean of its members (one bounded MacQueen/Lloyd-style step — over nights this converges toward the local k-means optimum while never jumping).
4. Detects an emptied galaxy explicitly and **retires** it (row kept, `retired_at` set, dropped from the public list, ID never recycled) — reachable precisely because no library relocation runs.
5. Writes back (write-order contract below).

IDs are stable _by construction_: no relabeling step exists, so no label-alignment machinery is needed at all. On an unchanged corpus the run is a no-op — identical assignments, centroids equal within 1e-6 cosine (float reductions vary with thread count even on one box, so the sweep pins its thread env, e.g. `OMP_NUM_THREADS=1`, and the invariant test uses that tolerance).

### The k policy: full fits are operator acts

- **Cold start (run 1 — explicitly a snap, not a ratchet):** one full `KMeans(n_init="auto", random_state=fixed)` fit at **k = 9 (ratified)** — the `clamp(round(sqrt(N/2)), 3, 24)` default at ~150 findings. Every cluster gets a server-minted ID + handle and enters the naming queue.
- **The split (operator-gated):** the admin view (Unit C) displays each galaxy's evidence — mean silhouette (coherence; the one O(N²) term, trivial at this scale, computed for display only) and member count vs the mean — so the operator can _see_ when one region holds two feels. The operator requests a split (`split_requested_at` via `update_galaxy`); the next nightly tick consumes it (the pulled-not-pushed queue discipline of `track-lifecycle.md`): a k=2 fit on that galaxy's members only, the parent's ID stays on the larger child, the Worker mints ONE new ID + handle for the smaller child, which lands unnamed in the naming queue. After any bulk embed backfill, the operational rule is: hold naming of new regions until the operator has split to taste — or remint before naming begins.
- **The remint (full reset):** never on the cron. An explicit operator-run invocation of the sweep with `--remint` on the box (documented in the operator runbook): a fresh full fit at the current `k` target, which **retires every old ID and sends every new cluster through the naming queue again** — names are deliberately orphaned, which is why it is manual. Reserved for a deliberate redraw: an embed-model change (a different checkpoint or `EMBEDDING_DIMS` stales every centroid) or a corpus shift too large for splits.

### Compute placement: the `fluncle-cluster` nightly box cron

A new deterministic `--no-agent` sweep on rave-02, cloned from the `fluncle-embed` shape (`docs/agents/hermes/scripts/embed-sweep.{sh,ts}` + `embed-track.py`; a new `cluster-timer/` unit dir is auto-discovered by `install-host-timers.sh`): `cluster-sweep.sh` → `cluster-sweep.ts` (bun orchestrator) → `cluster.py` (numpy for the nightly assignment step; sklearn + scipy for the operator-act fits), running in the image's baked MuQ venv (the Dockerfile's `MUQ_VENV`).

- **The venv layer bump is not a casual `pip install`.** The MuQ Dockerfile layer's own ordering law (the CPU torch trio in its OWN pip step, THEN muq — or the resolve drags in CUDA builds and the import fails) means sklearn/scipy go in as a **third pinned pip step with numpy constrained to the already-baked version**, and the pin-watch pre-smoke extends from `import torch, muq` to `import torch, muq, sklearn, scipy` so a broken rebuild can't swap in silently.
- **Not the Worker.** `get_similar_findings` is a per-request single-target scan, correctly in the Worker; the cluster engine is a stateful batch job that wants numpy/sklearn and the box's admin token. The Worker's role is the identity mint and the transactional map write (below).
- **Nightly, not every 5 minutes.** A nightly `.timer` (align with the audit-timer band; host clock UTC) is plenty for a browse surface and reinforces the fixed-point discipline. Zero LLM tokens; sub-second CPU (silhouette for the admin evidence is the term that eventually dominates — never "optimize" the assignment step for it); a few MB peak RAM (no RAM pilot needed, unlike MuQ's 2.85 GB).
- **What it reads:** the embedded corpus (`log_id IS NOT NULL AND embedding_json IS NOT NULL` — the `get_similar_findings` candidate set) via the **cursor-paginated** `list_track_embeddings` op (limit/offset over a table the embed cron mutates every 5 minutes skips or duplicates rows; the `decodeTrackCursor`/`encodeTrackCursor` machinery in `tracks.ts` used by `list_tracks_admin` is the precedent — key the cursor on `track_id`; a finding embedded mid-sweep simply joins the next night, benign by design), plus the prior map + `split_requested_at` flags via `list_galaxies_admin`.
- **The write-order contract (the run's real consistency boundary):** (1) upsert the map FIRST via `update_galaxy_map` — the Worker mints any new IDs/handles and returns the minted rows, so every ID a track can point at exists before any assignment lands; (2) write the _changed_ assignments (a handful per night) via the generic `update_track` path; (3) retire/refresh map rows in a final map write. `member_count` is NOT stored — it is derived (`COUNT(*) GROUP BY galaxy_id`) in the read ops, deleting the denormalization-drift class outright. A mid-run crash is resume-safe: re-running converges, and no assignment can ever point at a missing map row.
- **A note the "quiet write" story needs:** galaxy assignment stays out of `VISIBLE_FIELDS` (no `updated_at`/sitemap churn), but `updateTrack` unconditionally purges the finding's `/log` edge cache (`purgeLogCache`) — which is exactly what the galaxy prose clause needs to refresh. Quiet for the sitemap, fresh for the reader.
- **Registry + cost:** register a `cron.cluster` surface in `@fluncle/registry` (the `cron.embed` precedent) and emit a `record_cost` event per run (`subsidized` / on-box compute; `record_cost` + the `cost-emit.ts` helper already exist and `embed-sweep.ts` already uses them).

### Findings without an embedding

A finding whose capture/embed hasn't landed (`embedding_json IS NULL`) cannot be placed yet — `galaxy_id` stays NULL and it joins the map after `fluncle-embed` drains it. Every surface treats the missing field as silence (nothing renders), the present-only rule every enrichment field follows (`track-lifecycle.md` §Surfacing).

## 3. Unit B — schema, ops, and the write-back path

### Schema (drizzle; `bun run --cwd apps/web db:generate`; never hand-written SQL)

**A new `galaxies` table** — the first time galaxy identity lives in the database (today's four are a hardcoded constant in `apps/web/src/lib/galaxies.ts`, derived per-track from the dead vibe quadrants). The structural precedent is the `artists` table (`apps/web/src/db/schema.ts` ~L1074) for the slug-addressable public entity, with `artists.spotifyArtistId` (nullable `.unique()`) as the precedent for the nullable-unique columns — SQLite treats NULLs as distinct in UNIQUE indexes, so many unnamed galaxies coexist. No separate slug index: `.unique()` already creates one.

```ts
export const galaxies = sqliteTable("galaxies", {
  centroidJson: text("centroid_json").notNull(), // the 1024-d centroid, JSON array like embedding_json — the nightly assignment anchor
  createdAt: text("created_at").notNull(),
  handle: text("handle").notNull().unique(), // minted once at birth, server-side, via galaxySlug(id, attempt) — the permanent admin/CLI identity (the plan-handle precedent), never renamed, never public
  id: text("id").primaryKey(), // server-minted stable ID (e.g. `gal_<ulid>`; the shape is free — never recycling it is the invariant)
  name: text("name"), // operator-authored display name; NULL until named — unnamed galaxies are admin-only
  retiredAt: text("retired_at"), // set when the galaxy empties; retired rows never leave the table and never recycle their ID
  slug: text("slug").unique(), // frozen with the name; NULL until named (may adopt the handle if the operator likes it)
  splitRequestedAt: text("split_requested_at"), // the operator's split trigger; the nightly tick consumes it
  updatedAt: text("updated_at").notNull(),
});
```

**A new column on `tracks`:** `galaxyId: text("galaxy_id")` — nullable, the internal-grouping precedent of `embeddingJson`/`vibeX`/`vibeY`. Hard assignment (one galaxy per finding); a `track_artists`-style join table is the named escape hatch if soft membership is ever wanted — it is not now. Member counts are derived, never stored.

The migration lands as the next numbered `apps/web/drizzle/` file (+ meta), committed with the schema change, applied automatically in the Cloudflare build on deploy.

### Write-back (the generic agent update path, per `track-lifecycle.md` §The update path)

Per-finding assignment rides `updateTrack` (`apps/web/src/lib/server/track-update.ts`) exactly like `embedding`: a `TrackUpdate.galaxyId` key, the `if (update.galaxyId !== undefined)` SQL block, the CLI flag `fluncle admin tracks update <id> --galaxy-id <id>`. **Deliberately NOT in `VISIBLE_FIELDS`** (the `embedding` precedent — no lastmod churn), while the built-in `purgeLogCache` keeps the `/log` prose fresh. The map write (`update_galaxy_map`) is atomic via `db.batch(statements, "write")` — libSQL's one-implicit-transaction batch, the `push.ts`/`recordings.ts` precedent — so a crash can never half-apply the map; run-level consistency comes from the write-order contract in Unit A.

## 4. Unit C — naming and the operator gate

### The noun: galaxies (ratified — no new entity)

The operator's ruling: "We have Galaxy for this already." The sonic clusters inherit the name **galaxies** — the k=9 sound-derived, operator-named regions REPLACE the four vibe-bucket galaxies outright. No parallel vocabulary, no new cosmos noun, no disambiguation burden: the crew keeps the word they know, and the word finally means something real. The existing overload is simply the status quo, restated once: **the Galaxy** (capital-G, singular) is the whole of Fluncle; **the galaxies** are its named regions — previously four fictional quadrants, now the real map. LORE.md's frame survives intact (each finding "a star — a waypoint"; the galaxies are where the stars gather). The four retired names (Solar, Nebular, Lunar, Astral) return to the vocabulary pool — the operator may reuse any of them for a real galaxy whose sound earns it. (The "constellation" coinage an earlier draft of this RFC floated is dead, which also dissolves its collision with the admin findings-board layout of the same name — no rename needed anywhere.)

**The naming collision to fix in canon:** `track-lifecycle.md` says the clusters "will drive a browse-by-feel lens + the game's solar systems" — on **both** line 33 and line 76 — two names for one object, violating the one-operation-one-name convention (`docs/naming-conventions.md`). Ratified resolution: **galaxy is the single canon noun for the cluster object; the game renders a galaxy as a spatial region, it does not rename it.** This build updates both lines.

### Replacing the four quadrant galaxies (ratified)

- **Not nested:** placing a sound-derived cluster in an energy×mood quadrant requires exactly the mood placement audio provably cannot learn (the n=45 signal test). Rejected on evidence.
- **Replaced:** `apps/web/src/lib/galaxies.ts` (the `GALAXIES` constant + `galaxyForVibe`) and the vibe-quadrant derivation (`galaxyOf`) retire; galaxy identity moves into the `galaxies` table. Because the name carries over, most readers keep their surface and change only their value source (the per-reader table in Unit E).
- **Continuity for the crew:** every embedded finding — including the ~45 old vibe-placed ones — gets a real galaxy from the cold-start fit, so no surface loses data; they all gain it.

### Authoring: the machine mints a handle, the operator names

The persona law (`docs/admin-shell.md`) and the note-agent precedent (fill-empty-only; the operator's word is never clobbered) shape the flow:

1. Each new galaxy is born with a permanent **handle**, minted server-side inside `update_galaxy_map` via `galaxySlug(id, attempt)` from `packages/contracts/src/util/galaxy-slug.ts` (`adjective-cosmos-scenenoun`, curated allow-list, `BANNED`-guarded, Banger Budget enforced, collision-salted — the signature already takes `attempt`; the util's name was always waiting for this job). The handle is the admin/CLI identity (the plan-handle precedent — minted once from a seed and stored), not a name proposal; naming is a purely editorial act the machine never attempts, and **a machine handle never renders publicly (ratified)**. No LLM anywhere; the sweep stays `--no-agent` deterministic.
2. The unnamed galaxy appears in the `/admin/galaxies` attention queue (AdminShell placement contract), identified by its handle, its member covers, and its evidence numbers. It is **not public** until named.
3. The operator names it — sets `name` + `slug` via `update_galaxy` (operator tier) **after listening**: the admin view carries an in-place audition (play member previews via the existing `/api/preview/:idOrLogId` relay) so naming nine clusters doesn't mean opening ninety `/log` pages. Naming mints the public URL. In-voice candidates for the spike's k=3 families: _The Liquid Deep_, _The Feral Steppers_, _Drifting Aurora_ (more in the appendix — names are editorial and need not be slug-pool words; the four retired quadrant names are also available).
4. **A re-run may move findings; it never renames a galaxy.** The name is decoupled from the drifting math — the minted-once-and-stored law that fixed the `predictedMixtapeLogId` drift bug (`galaxy-slug.ts` header) and the opaque-coordinate law behind the Log ID (VOICE.md).

The admin surface is one AdminShell view: the galaxy list (name/handle, member covers + derived count, the audition, the evidence columns), the name/rename control, the split trigger, and the attention queue for unnamed ones. The remint stays a deliberate on-box operator invocation, not a button.

## 5. Unit D — the public lens

### Routes

`/galaxies` (index) + `/galaxies/$slug` (one galaxy) — files `apps/web/src/routes/galaxies.index.tsx`, `galaxies.$slug.tsx`. **The game keeps `/galaxy` and galaxy.fluncle.com** (the registry's `web.galaxy` + `subdomain.galaxy`); the plural detail route is deliberate — the artists precedent's singular detail (`/artist/$slug`) would put the lens at `/galaxy/$slug`, nesting under the game's route, so `/galaxies/$slug` it is (ratified; no better shape found). SSR-first `createServerFn` loaders reading the shared `apps/web/src/lib/server/galaxies-map.ts` functions, `Route.useLoaderData()` — the public-archive pattern, not the admin react-query pattern. Canonical `createFileRoute` property order with the `// oxlint-disable-next-line sort-keys` escape where loader precedes head, per AGENTS.md; `/galaxies/$slug` takes the `log.$logId.tsx`-style params → beforeLoad → loader → `notFound()` shape.

### Register and shape (the canon rails)

- **Never chips.** The anti-chip stance is synthesized from three real rules — DESIGN.md §Don't ("SaaS dashboards, bright streaming-app clones, or generic landing-page hero sections"), the VOICE.md web register ("an archive of findings you browse, not a feed you scroll"), and the genre argument in `track-lifecycle.md` ("an absolute label is an argument; a relative placement is answerable"). A galaxy is a _place you enter_, not a facet you toggle.
- **The index** is the map: each galaxy as a quiet, cover-led pane — member cover tiles under the name set in the **Title register (system sans, extrabold cream — the artist-name treatment)**, with the numeric `N findings` line in Oxanium tabular per the Tabular Rule. ## Acceptance criteria

- A new finding lands in a galaxy within one nightly tick of its embedding landing; zero manual steps in the assignment loop.
- The fixed-point invariant: two consecutive runs over an unchanged corpus produce identical assignments and centroids within 1e-6 cosine (thread env pinned); a run with +5 synthetic findings moves no existing assignment except genuine boundary cases.
- The empty-retire path: a fixture whose galaxy empties gets `retired_at` set and keeps its ID out of circulation — no teleport, because no full fit runs on the cron.
- An operator split request is consumed on the next tick: the parent keeps its ID, exactly one new server-minted ID + handle appears, the child lands unnamed in the queue. A remint retires every old ID and re-queues the whole map for naming.
- The write-order contract holds under a mid-run crash (resume-safe test): no assignment ever points at a missing map row; member counts are derived, never drift.
- All six ops pass the four coverage tests (`orpc-coverage`, `orpc-admin-coverage`, `orpc-auth-coverage`, `orpc-naming`); `update_galaxy` 403s an agent token (operator tier); the map write is one `db.batch` transaction; the `me-galaxy` game-save ops are untouched.
- An unnamed galaxy is invisible on every public surface (its handle never renders publicly) and queued in admin with audition + evidence; naming mints the public URL; a re-run never changes a name, slug, or handle.
- `/galaxies` and a named `/galaxies/$slug` render SSR with `ItemList`/`MusicPlaylist` JSON-LD, appear in `sitemap.xml`, order members core-first (paginated per the contract), and pass the canon-reviewer rails (no chips, names in the Title register not Oxanium, One Sun, dark-only, AA contrast, keyboard access, reduced-motion); the game's `/galaxy` route and galaxy.fluncle.com are unaffected.
- The `/log` page carries the galaxy in the prose AND the `MusicRecording` JSON-LD description (the log-prose mirror), now linking to `/galaxies/<slug>`, shows nothing for an unplaced finding, and the galaxy related row is gone — in one release, no interim double state.
- `fluncle galaxies` (CLI) and the SSH galaxies screen both list and open galaxies; the registry surfaces are registered and `/status` probes green; the box image pre-smoke imports `torch, muq, sklearn, scipy`.
- The reader swap is complete per the Unit E table (log-prose, OG, radio, CLI placement, agent-discovery, editions matcher, the DTO type change, the related-row removal, `lib/galaxies.ts` + the `Galaxy` union retired, the full vibe write-path retirement incl. `VISIBLE_FIELDS`/`OPERATOR_ONLY_FIELDS`/handler/CLI); the ~45 vibe-tagged findings render their NEW sound-derived galaxy everywhere.

## Risks & open questions

- **k = 9 is ratified, but the families still have to feel right.** The spike validated k=3 on 54 tracks; the pilot exists to let the operator hear the 9-way split before naming begins — and the split/remint acts are the levers if the corpus outgrows it.
- **Old placed findings change galaxy.** The ~45 vibe-tagged findings will re-file under sound-derived galaxies whose names differ from the quadrant they used to show. This is the point (the old placement was the fiction), but the newsletter's past editions referenced the old names — past editions are sent artifacts and stay as-is; only the matcher for future editions retargets.
- **The DTO type change is a public API break.** `/api/tracks`' `galaxy` field goes from a four-name string to a nullable `{ name, slug }` object. The known consumer (the newsletter matcher) updates in the same delivery; unknown external consumers of the public API see a changed field — acceptable for a field that has been silently absent for every new finding since tagging retired, but it ships with a note in the API docs/OpenAPI description.
- **Embed-model coupling.** The centroid map is valid only in the current MuQ space; any future embed-model/pooling change (a different `EMBEDDING_DIMS` or checkpoint) stales the map and MUST trigger the operator remint. Bake this warning into the maintenance notes for the MuQ Dockerfile layer.
- **The cold-start naming session.** The launch gate means one operator sitting to name 9 clusters (with audition support). Splits thereafter arrive only when the operator asks for them, so there is no ongoing naming debt.
- **Assignment-only drift.** One mean-update step per night tracks gradual corpus drift but never restructures; after a bulk backfill the map can lag the corpus's true shape until the operator splits or remints (the stated operational rule). This is the deliberate trade: structure changes are human acts.
- **Per-galaxy color** is deliberately unshipped; if a surface (the OG card, the game) later wants one, it is a `color` column + one migration.
