# The admin jobs inventory

[`admin-jobs.csv`](./admin-jobs.csv) is the working document for the workflow-centralization epic (2026-07-06): every job-to-be-done in the operator's system — live, partial, and desired — time-ordered from "a track got added", grouped by domain. It replaces a brief/RFC on purpose: migrate, clean up, polish, add, verify, and test against rows, assigning a sub-agent per domain or per job.

Columns: `domain` · `order_hint` (pipeline position within the domain) · `job` · `status` (live/partial/desired) · `trigger` (operator/cron:name/event:what/agent) · `executor` (human/ai/system) · `runs_on` (cloudflare/rave-01/2/3/M2/M5/phone/browser) · `input` · `output` · `surfaces` (which can invoke it today) · `primary_surface` (where it SHOULD live per the persona law: web admin = the operator's platform, `fluncle admin` = the agents' tool, `fluncle` = the crew's toy) · `drop_from` (surfaces to retire it from — the de-fragmentation column; `helm(frozen)` marks the frozen app's candidates) · `tests` (yes/partial/none) · `notes` (code anchors).

Snapshot at creation: 126 jobs — 99 live, 9 partial, 18 desired; test coverage 35 yes / 41 partial / 50 none; 18 rows carry drop candidates. Headline findings: no database backup/restore path exists; the TikTok stale-draft trap silently marks bounced drafts as posted; the web admin can neither add a finding nor touch the video pipeline; `cron.studio-clip` runs invisibly (unregistered, unmonitored); announce + MusicBrainz + Wikidata are fully manual; browser multipart upload is one UI component away.
